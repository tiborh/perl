Packet Core Product Configuration and Integration Service Delivery Instruction

The Global Competence Center (GCC) for PC is GSDC Spain. See PC GCC portal.

1. Introduction

This document gives directives and guidelines for the delivery of the Product Configuration and Integration Services for network elements in the Packet Core (PC) domain. The most important aspects impacting the delivery are explained, e.g. service impact for a specific network element and release, relations to other services, methods and tools.


This document is intended for persons in project organizations, engineers and project managers. Line management can also make use of the document.

Suggestions and comments on this document should be sent to the author of the document. For more information on the organization for Product Management of PA Consulting & Systems Integration (CSI), Core Network or/and ETXFREH

2. Generic information

The PC domain includes all packet switching nodes in the Ericsson portfolio: CGSN, SGSN, SGSN-MME, SGSN in pool, SPP, GGSN, GGSN-MPG, CPG, SASN and SAPC. For the different releases of the products listed above, the generic Product Configuration and Integration delivery information in this document is to be used. See chapter 2.1.

This SDI for Product configuration and Integration is related also Service Deployment Guide for Packet Core Network. The product configuration and integration services are among the Ericsson Global Services portfolio.

- For more details on the Product Configuration Service, see Data Collection and Configuration

- For the Integration Service, see Data Collection and Configuration

- For more information on the Product Configuration process, see Product Configuration Process Description.

- For more information on the Integration process, see Integration Process Description.

- See Life Cycle dates and Packet Core Roadmap for PC nodes.

- See Compatibility Matrix for compatibility information between PC products and related solutions.

- Find more information regarding Ericsson Packet Core Product Services inside the CSI portal

The general process information is complimented in the specific SDI documents for Packet core network elements which include more details supporting the generic process, like how to find the PCI manual.

2.1. Support during integration:

Working with the Configuration and Integration of the node there are several ways to get support. There are knowledge sharing communities covering Packet Core/SACC integration which are found under chapter 6 in this SDI. There is also a possibility to look for known issues already reported as Customer Service Requests (CSR).  This can be done through the KCS - Knowledge-Centered Support  Just to be able to look for known issues in the KCS there is a read only entry point to the data found through the KCS link.  

Customer Service Requests (CSRs) that are issued by an Integration project before customer acceptance and handover to Support will be charged back to the projects at a fixed cost per CSR. These CSRs are also known as “project CSRs”. Please see Global Support During Integration and Support During Integration Cost 2010 for details.

2.2 BENT Reporting

BENT is used to report planned update and upgrade activities. All Integration activities which could result in outages must be registered in BENT. See Upgrade/Update Event Reporting - Directive for details on when to report coming updates and/or upgrades. All upgrades or updates should be reported in BENT. Even unplanned outages should be reported retroactively.

2.3 Upgrades

The Product Development Unit (PDU) Packet Core provides upgrade instruction valid for that particular SW. In case configuration updates are required as part of the SW upgrade/update, information on how to handle this will be included in the upgrade instruction. These upgrade instructions are available at PLM GSN’s SW Deliveries.

 

2.4 Competence

- The functional roles (word/html) and competence (functional skills - FS) needed for PC Consulting & System Integration are according to the ones indicated in Functional Skills for PC, Functional Skills for SACC, Functional Skills for SmartEdge and Functional Skills for EPC

- Ericsson Academy has been created for the needs of internal competence build up and for the Packet Core area the first area covered is the EPC which can be found at Ericsson Academy PC/EPC

- PDU competence build up can be found in PDU PC Training, EPG (Evolved Packet Gateway) Training, Packet Core NIV test and SASN/SAPC Competence Build Up

2.5 Remote integration

All PC nodes can be managed remotely, meaning that all SW and configuration data handling, integration and verification can be performed off site from e.g. the customer’s Network Operation Center (NOC). If the customer has a secure system allowing for external access to the O&M domain where the node resides, e.g. Remote Support Gateway (RSG), the above listed activities can be performed from Ericsson premises (MU, GSDC or other). Please note that the Ericsson premises can be extended to include the integration engineer’s home when using RACOM via Internet (RVI).

If the RSG is to be used for remote service delivery, it is important to note that permission to remotely log into customer’s nodes must be granted by the local Ericsson organization.

3. Service Product Delivery Information. Delivery tools

Three tools are available today to be used during PC Network Planning and Design:

›          CANDI: used for PC node dimensioning and capacity evaluation

›          COUNT: recommended for network planning, scenario evaluation, contingency plan and routing analysis

›          CCR: used for PC node configuration; it is the interface between PC Network Design and Product Configuration

All these tools can be selected at Tools portal.
